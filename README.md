# Алгоритм быстрой сортировки

Алгоритм быстрой сортировки

Синтетический тест быстроты сгенерированного кода разных компиляторов.

Требования:

1. Алгоритм быстрой сортировки (сортировка Хоара).
2. Сортировка по возрастанию: от маленького значения к большему.
3. Функция сортировки либо написана самостоятельно, либо находится в стандартной библиотеке. Посторонние библиотеки не использовать.
4. Не использовать многопоточность.
5. Длина массива 50000000 (пятьдесят миллионов).
6. Тип данных для сортировки: структура из двух полей типа Double: LowPart и HighPart.
7. Функция сравнения: сравниваются HighPart, затем LowPart.
8. Десять циклов измерений времени, выводится среднее.
9. На каждом цикле массив заполняется случайными значениями в диапазоне `[0, 1)`.
10. Измеряется время сортировки массива.
11. Затравка для генератора случайных чисел равна нулю.
12. Генератор случайных чисел инициализируется заново перед каждым заполнением массива случайными числами, чтобы количество вызовов функции сотировки было одинаковым на каждой итерации цикла.
